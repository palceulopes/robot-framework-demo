# Map: convention vs customization

Quick mental model for interviews and maintenance:

- **Platform / convention**: follow uv, PEP 621, Robot, Flask, paho docs‚Äîyou do not owe a line-by-line defense.
- **Contract / domain**: MQTT topics, REST routes, CAN/DBC signals, test scenarios‚Äîwhere engineering judgment shows.
- **Mixed**: same file‚Äîpart idiomatic boilerplate, part your design.

```mermaid
flowchart LR
  subgraph platform [Platform]
    uv[pyproject_uv_lock]
    git[gitignore]
    robot_meta[robot_yaml]
  end
  subgraph domain [Demo domain]
    cfg[variables_config]
    mocks[mock_servers]
    libs[libraries_domain_logic]
    tests[tests_robot]
    dbc[vehicle_signals_dbc]
  end
  platform --> domain
```

## Mostly platform / convention

| File | Why |
|------|-----|
| [`uv.lock`](../uv.lock) | Generated by `uv`; do not hand-edit; purpose is reproducible installs. |
| [`.gitignore`](../.gitignore) | Common Python template + Robot outputs (`report.html`, `output.xml`, `results/`) + `.venv`; explicit choices mainly in the final ‚ÄúProject specific‚Äù block. |
| [`pyproject.toml`](../pyproject.toml) | **PEP 621**: `[project]`, `dependencies`, `optional-dependencies`. **`[build-system]`**: PEP 517 default. **`[tool.black]` / `[tool.mypy]`**: typical team/IDE settings. **`[tool.setuptools.packages.find]`**: packaging layout (`libraries*`, `variables*`), not automotive domain. Interview line: ‚ÄúI declare ranges and extras here; the lockfile pins exact versions.‚Äù |

## Mixed files (same file: ‚Äúthat‚Äôs just how it works‚Äù vs ‚Äúyour design‚Äù)

### [`libraries/mqtt_vehicle_network.py`](../libraries/mqtt_vehicle_network.py)

- **Convention**: `logging.getLogger(__name__)`, optional `ImportError` guard, `@library(...)`, `mqtt.CallbackAPIVersion.VERSION2`, connect/subscribe/publish flow per paho docs.
- **Customization**: `TOPICS` mapping (`vehicle/sensors/...`), exposed keywords, JSON payloads.

### [`libraries/rest_ecu_api.py`](../libraries/rest_ecu_api.py)

- **Convention**: `requests.Session`, `urllib3.Retry` + `HTTPAdapter`‚Äîresilient HTTP client pattern from `requests` docs.
- **Customization**: `ENDPOINTS`, `get_ecu_signal` naming, default `base_url`‚Äî**contract with** [`mock_servers/ecu_rest_server.py`](../mock_servers/ecu_rest_server.py).

### [`libraries/automotive_lib.py`](../libraries/automotive_lib.py)

- **Convention**: `@library`, logging, `@dataclass`, `cantools` / `python-can` as documented.
- **Customization**: `AdbMock` properties/commands, `CanBusManager` and `send_signal` (full-frame fill), [`DBC_PATH`](../variables/config.py) integration.

### [`libraries/automotive_listener.py`](../libraries/automotive_listener.py)

- **Convention**: `ROBOT_LISTENER_API_VERSION = 2`, `start_suite` / `end_test` / ‚Ä¶‚Äî**Robot listener API**.
- **Customization**: counters (`signal_count`, `api_call_count`, ‚Ä¶), `logs/` output, JSON metric shape‚Äî**your observability story**.

### [`mock_servers/mqtt_broker_helper.py`](../mock_servers/mqtt_broker_helper.py)

- **Convention**: `amqtt.broker.Broker`, asyncio, signal handling‚Äîembedded broker script pattern.
- **Customization**: logging messages, CLI surface (if any)‚Äîdemo policy.

### [`setup_project.py`](../setup_project.py)

- **Convention**: `logging.basicConfig`, `sys.version_info`, dynamic imports for dependency checks.
- **Customization**: `required_files` / directories‚Äî**your definition of ‚Äúminimum viable project‚Äù.**

### [`quickstart.py`](../quickstart.py), [`delete_logs_results.bat`](../delete_logs_results.bat)

- **Convention**: `subprocess.run`, `Path(__file__).parent`; batch `rmdir` for cleanup.
- **Customization**: [`quickstart.py`](../quickstart.py)‚Äîcommands you wrap. [`delete_logs_results.bat`](../delete_logs_results.bat) is **only** local cleanup of `results/`, `logs/`, `.robocache/` (no `uv`); run tests from the shell with `uv run robot ‚Ä¶`.

Robot settings use `Variables    variables.config` (same idea as `import variables.config`): requires `--pythonpath .` or editable install so the `variables` package resolves.

### [`robot.yaml`](../robot.yaml)

- **Convention**: Robot config format (`*** Settings ***`), standard libraries (`Process`, `String`, ‚Ä¶), `Test Timeout`, `Suite Concurrency`.
- **Customization**: values (e.g. 60s, concurrency 1) and which libs you preload.

### [`libraries/__init__.py`](../libraries/__init__.py)

- **Convention**: `__all__`, `__version__`, re-exports‚Äînormal Python package.
- **Customization**: **which** symbols you expose as public API.

### [`variables/__init__.py`](../variables/__init__.py)

- Negligible for interviews (short docstring only).

## Intra-file patterns you can label ‚ÄúPython/Robot idiom‚Äù (no deep story)

- `logger = logging.getLogger(__name__)`
- `from robot.api.deco import library` + `@library(...)` (Robot Framework 7 style libraries)
- `ROBOT_LISTENER_API_VERSION` (listener API requirement‚Äîone sentence is enough)
- Flask: `Flask(__name__)`, `CORS(app)`, `@app.get`
- Long docstrings in libs‚Äîoral shortcut: ‚Äúthe keyword layer documents intent.‚Äù

---

## Not ‚Äúyour code‚Äù in an interview sense

- Contents of `.venv`, `*.egg-info`, `site-packages`‚Äîinstall artifacts and dependencies.

## One-liner: integration contract

**Central configuration lives in `variables/config.py`; the mock ECU defines REST routes and JSON shapes; Robot resources map readable keywords onto those contracts‚Äîso endpoints and brokers can change via config without rewriting scenarios.**

## Interview phrase (professional tone)

‚ÄúThe dependency and packaging layers follow standard Python and uv practice; where I invested judgment is the contract between the mock ECU, the MQTT topics, and the Robot keywords‚Äîso we can change endpoints or brokers via config without rewriting scenarios.‚Äù
